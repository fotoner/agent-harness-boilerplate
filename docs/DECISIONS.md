# Architecture Decision Records (ADR)

프로젝트의 주요 기술적 결정을 기록한다. 에이전트가 "왜 이렇게 돼있지?"를 물을 때 여기를 참조한다.

## ADR 작성 규칙

1. **결정이 내려질 때마다** 새 ADR을 추가한다 (아키텍처, 기술 선택, 라이브러리, 패턴 등)
2. 한 번 승인된 ADR은 **수정하지 않고**, 새 ADR로 대체한다 (이력 보존)
3. `/office-hours` 디자인 문서나 `/plan-eng-review` 결과에서 결정이 나오면 여기에 요약을 남긨다

## 템플릿

```markdown
## ADR-{NNN}: {제목}

- **상태**: 제안 / 승인 / 대체됨 (by ADR-{NNN})
- **결정일**: YYYY-MM-DD
- **맥락**: 어떤 문제를 해결하려 했는가?
- **결정**: 무엇을 선택했는가?
- **근거**: 왜 이 선택을 했는가?
- **대안**: 검토했지만 선택하지 않은 것들
- **결과**: 이 결정으로 인해 달라지는 것
```
